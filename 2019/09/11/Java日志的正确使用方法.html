打日志的正确方式

什么时候应该打日志

  1. 当你遇到问题的时候，只能通过debug功能来确定问题，你应该考虑打日志，良好的系统，是可以通过日志进行问题定为的。
  2. 当你碰到if…else 或者 switch这样的分支时，要在分支的首行打印日志，用来确定进入了哪个分支
  3. 经常以功能为核心进行开发，你应该在提交代码前，可以确定通过日志可以看到整个流程
